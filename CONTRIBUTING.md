# Contributing Guide

Thank you for your interest in contributing to this project. Contributions are welcome and appreciated.

This project follows a simple, practical workflow to keep contributions clean, reviewable, and consistent.

---

## How to Contribute

### 1. Fork the Repository
Click the **Fork** button on GitHub to create your own copy of the repository.

---

### 2. Clone Your Fork
```bash
git clone https://github.com/<your-username>/container-orch.git
cd container-orch
````

---

### 3. Create a Feature Branch

Always create a new branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

---

### 4. Make Your Changes

* Follow existing code style
* Keep changes focused (one feature or fix per branch)
* Avoid unrelated refactoring

---

### 5. Test Your Changes

Make sure your changes do not break existing functionality.

If tests exist:

```bash
make test
```

Or run manually depending on the component.

---

### 6. Commit Your Changes

Use clear and meaningful commit messages.

```bash
git commit -m "feat: add new feature description"
```

Common prefixes:

* `feat:` new feature
* `fix:` bug fix
* `docs:` documentation changes
* `chore:` maintenance tasks

---

### 7. Push Your Branch

```bash
git push origin feature/your-feature-name
```

---

### 8. Open a Pull Request

Go to the original repository and open a Pull Request.

Include:

* What you changed
* Why you changed it
* Any relevant context

---

## Code Guidelines

* Keep code readable and simple
* Avoid committing secrets or credentials
* Follow existing project structure
* Ensure backward compatibility when possible

---

## Reporting Issues

If you find a bug or security issue, open a GitHub issue with:

* Clear description
* Steps to reproduce
* Expected vs actual behavior

---

## Want to Suggest Features?

Feature ideas are welcome. Open an issue and label it as `enhancement`.
